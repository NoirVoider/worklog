# Worklog Sidebar 动效与自动保存设计

## 背景

当前主窗口的侧栏折叠/展开通过 `width` 和 `min-width` 动画驱动，主编辑区会在动画过程中持续参与重排。实际体感是：

- 侧栏不是“平移出来”，而是像被裁切和挤压。
- 主内容区会跟着抖动式重排，容易掉帧。
- 侧栏内部内容有延迟淡出，视觉上会出现“半截内容被吞掉”的过程。

编辑区目前仍然使用显式保存按钮。这个交互与常见编辑器的默认体验不一致，也会让“保存”成为一个持续存在的视觉负担。

## 目标

- 让侧栏具备“从左侧平移出来”的体感，不再通过布局重排驱动动画。
- 保持侧栏和折叠按钮常驻 DOM，避免因为挂载/卸载带来的顿感或状态丢失。
- 去掉保存按钮，改为接近 `files.autoSave = afterDelay` 的自动保存体验。
- 保证自动保存失败时不丢内容，切日期和关闭窗口时也能尽量安全落盘。

## 非目标

- 本次不引入用户可配置的自动保存延迟。
- 本次不改动日记文件格式，也不改动 Rust 后端命令接口。
- 本次不为侧栏增加遮罩层、拖拽缩放或多级导航结构。

## 决策摘要

采用以下组合方案：

1. 侧栏改成绝对定位的左侧独立层，固定宽度，始终挂载。
2. 展开/折叠仅通过 `transform` 和少量 `opacity` 驱动，不再动画 `width`、`min-width`。
3. 保存按钮移除，编辑区改为 `afterDelay` 自动保存，默认延迟约 `600ms`。
4. 在切日期、窗口失焦、窗口关闭和 `Cmd/Ctrl+S` 时执行显式 flush。
5. 为读写请求加“最新请求保护”，避免旧请求回写覆盖新内容。

## 当前实现问题

### Sidebar

- `.sidebar` 当前使用 `width: 288px` 到 `0` 的过渡。
- `.sidebar-body` 使用延迟 `opacity` 过渡，与宽度裁切叠加后会放大卡顿感。
- `.sidebar-titlebar` 和 `.sidebar-body` 仍保留 `min-width: 288px`，折叠时更像内容被强行压缩而不是整体平移。
- 折叠态的展开按钮是条件渲染，状态切换时存在额外的挂载/卸载。

### 保存模型

- 编辑内容完全保存在 React state 中，只有点击按钮或快捷键时才调用 `saveEntry`。
- 切日期会直接读新内容，如果当前日期尚未保存，存在用户误以为已经写入的风险。
- 当前 `isSaving` 与 `status` 只描述“手动保存按钮”的状态，不适合表达自动保存流程。

## 方案设计

### 一、Sidebar 结构与动画

#### 结构调整

- `app-shell` 从“侧栏列 + 主内容列”的网格布局改成单层相对定位容器。
- `main-surface` 永远占满窗口，不再受侧栏开关影响。
- `sidebar` 改为左侧绝对定位层：
  - 固定宽度 `288px`
  - 纵向铺满窗口
  - 始终挂载在 DOM 中
- 展开按钮保持为 `sidebar` 的同级元素，常驻 DOM，仅通过样式控制显示与交互性，避免跟随侧栏一起移出屏幕。

#### 动画原则

- 展开态：`transform: translate3d(0, 0, 0)`
- 折叠态：`transform: translate3d(-100%, 0, 0)`
- 可辅以轻量 `opacity` 变化，但不再动画以下属性：
  - `width`
  - `min-width`
  - `border-right`
  - `box-shadow`
  - `background`
- 推荐节奏：
  - 展开 `220ms cubic-bezier(0.22, 1, 0.36, 1)`
  - 折叠 `180ms cubic-bezier(0.4, 0, 0.2, 1)`

#### 交互约束

- 侧栏内部日历和最近日记列表始终保留，不做条件渲染。
- 折叠时通过 `pointer-events: none` 阻止隐藏层误触。
- 展开浮钮使用 `opacity/visibility/pointer-events` 控制，而不是 JSX 条件分支。
- 增加 `prefers-reduced-motion` 分支：用户系统要求减少动画时，侧栏直接瞬时开关。

### 二、自动保存模型

#### 目标语义

编辑体验参考编辑器常见的 `files.autoSave = afterDelay`：

- 用户输入停止约 `600ms` 后自动保存。
- 去掉显式保存按钮。
- `Cmd/Ctrl+S` 保留，但语义变成“立即 flush 当前待保存内容”。

#### 触发时机

- 输入停止达到延迟阈值时触发自动保存。
- 切换日期前先 flush 当前日期内容。
- 窗口失焦时 flush 一次。
- 主窗口关闭请求时 flush 一次，完成后再允许关闭。
- 手动快捷键 `Cmd/Ctrl+S` 时立即 flush。

#### 脏状态定义

- `content !== lastPersistedContent` 视为 dirty。
- `lastPersistedContent` 不再完全依赖 `currentEntry.content`，而是由成功写盘后的最新快照维护。
- 自动保存失败时：
  - 不回退编辑框内容
  - dirty 状态继续保留
  - 下次输入或快捷键继续重试

#### 空白日期

- 当某一天尚未创建文件时，允许直接开始输入。
- 自动保存直接调用 `saveEntry(date, content)`，由后端负责创建目录与文件。
- “创建”提示条本轮保留，但只作为信息提示和快捷入口存在，不再是写入链路的必要前置步骤。

### 三、并发与一致性保护

自动保存要重点规避两类问题：旧保存覆盖新内容、旧读取覆盖新日期。

#### 保存保护

- 每次保存都基于当前的 `date + content` 生成一次快照。
- 为保存请求分配递增的 request id。
- 只有最新且仍然匹配当前文档上下文的保存结果，才能更新：
  - `lastPersistedContent`
  - `currentEntry`
  - `status`
- 如果保存进行中用户又继续输入，不打断当前请求；请求结束后若内容仍然 dirty，则立即补一次新的自动保存。

#### 读取保护

- `loadEntry(selectedDate)` 同样使用递增的 request id。
- 只有最后一次发出的读取请求，才允许写回编辑区状态。
- 如果用户快速连续切日期，较慢返回的旧请求必须被忽略。

#### 切日期保护

- 用户切换日期时先对当前日期执行 flush。
- flush 成功后才更新 `selectedDate` 和 `viewDate`。
- flush 失败则留在当前日期，并通过状态栏或错误条明确反馈失败。

### 四、状态展示

状态栏继续保留，但含义从“按钮状态”切换为“自动保存状态”。

建议状态枚举：

- `就绪`
- `读取中`
- `未创建`
- `未保存更改`
- `自动保存中`
- `已自动保存`
- `自动保存失败`

状态更新原则：

- 输入发生后立刻进入 `未保存更改`
- 自动保存请求发出时进入 `自动保存中`
- 成功后进入 `已自动保存`
- 失败后进入 `自动保存失败`
- 如果内容再次变化，则重新回到 `未保存更改`

## 测试计划

### 样式与结构

- 断言 `.sidebar` 不再使用 `width/min-width` 动画。
- 断言折叠/展开改为 `transform` 驱动。
- 断言保存按钮不再出现在主编辑区。

### 自动保存行为

- 使用 fake timers 验证输入停止 `600ms` 后会调用 `saveEntry`。
- 验证连续输入时只保存最终快照，不会每次击键都写盘。
- 验证 `Cmd/Ctrl+S` 会立即触发 flush。

### 边界行为

- 验证切日期前会先 flush 当前内容。
- 验证 flush 失败时不会切走当前日期。
- 验证自动保存失败后内容仍保留、dirty 状态仍存在。
- 验证旧 `readEntry`/`saveEntry` 返回不会覆盖较新的内容状态。

## 风险与缓解

### 风险 1：关闭窗口时保存超时导致关闭手感变差

缓解：

- 关闭前只等待一次当前 flush，不做重复重试。
- 保持保存逻辑短路径，尽量复用已经准备好的最新快照。

### 风险 2：自动保存过于频繁

缓解：

- 使用固定延迟去抖，默认约 `600ms`。
- 保存中如果有新输入，只在当前请求完成后补一次，而不是并发堆积。

### 风险 3：状态文案过于频繁跳动

缓解：

- `已自动保存` 在状态栏保留约 `1.5s` 后回落到较安静的默认态，不额外弹出 toast。
- 错误态优先级高于成功态，直到下一次成功保存为止。

## 实施顺序

1. 重构 sidebar 结构，先让布局脱离 `grid` 推挤。
2. 用 `transform` 完成侧栏与浮钮动画，并补 reduced motion。
3. 引入自动保存状态与去抖逻辑，移除保存按钮。
4. 接入窗口失焦、切日期前 flush、关闭前 flush。
5. 补齐样式与行为测试，确保旧请求保护成立。

## 验收标准

- 侧栏展开/折叠的主观体验接近“整体从左侧平移进出”。
- 主编辑区在侧栏开关时不再随着动画重排。
- 不需要点击保存按钮即可稳定落盘。
- 快速输入、切日期、失焦、关闭窗口时都不会明显增加丢字风险。
- 自动保存失败时，用户编辑内容依旧完整保留并可继续重试。
